1. Ringkasan
Dokumen ini merangkum kebutuhan pengembangan website profil dan informasi desa sebagai salah satu program kerja (proker) KKN. Website ini dikembangkan berdasarkan hasil riset terhadap website desa-desa lain, dengan tujuan menyediakan informasi profil desa, data infografis, katalog UMKM/wisata, berita, dan dokumen publik (PPID) secara terpusat dan mudah diakses oleh warga maupun pihak luar.
Pengembangan dilakukan oleh 2 orang secara paralel: satu orang menangani frontend, satu orang menangani backend, dengan pendekatan contract-first agar integrasi antar bagian berjalan lancar.
2. Tujuan
Menyediakan informasi profil, sejarah, dan struktur organisasi desa secara digital.
Menyajikan data infografis desa (penduduk, APBDes, stunting, bansos, IDM, SDG) secara visual dan mudah dipahami.
Mempromosikan UMKM dan destinasi wisata desa melalui katalog interaktif berbasis peta.
Menyediakan kanal publikasi berita dan dokumentasi kegiatan desa.
Memenuhi keterbukaan informasi publik (PPID) melalui bank dokumen yang terkategorisasi.
Menyediakan kanal pengaduan warga yang mudah diakses tanpa mengganggu pengalaman pengguna.

5. Entity Relationship Diagram (ERD)
ERD berikut menggambarkan skema database awal yang mencakup seluruh modul: profil desa, berita, dokumen PPID, katalog UMKM/wisata, galeri, kegiatan, pengaduan, dan data infografis.
Catatan: nama field dan tipe data pada ERD di atas masih dapat berubah menyesuaikan hasil diskusi dengan perangkat desa, khususnya pada modul PPID dan infografis.
6. Strategi Implementasi Bertahap
Desain skema (ERD) disepakati secara menyeluruh di awal untuk menghindari design flaw pada relasi antar tabel. Namun implementasi/coding dilakukan bertahap per fase agar progres mudah diukur dan diuji end-to-end.
Fase 0 – Desain
Finalisasi ERD & skema Prisma untuk seluruh entitas.
Fase 1 – Foundation
USER (autentikasi admin)
VILLAGE_PROFILE & STRUKTUR_ORGANISASI
Fase 2 – Modul Independen
NEWS (+ admin CRUD)
DOCUMENT_CATEGORY & DOCUMENT (PPID)
KATALOG_CATEGORY, KATALOG, GALERI, KEGIATAN
PENGADUAN
Fase 3 – Modul Infografis
PENDUDUK_STAT, APBDES, IDM_SDG_SCORE, STUNTING_BANSOS
Ditempatkan di akhir karena data source masih menunggu diskusi lanjutan dengan perangkat desa
Setiap fase mengikuti alur: migrate tabel → seed dummy data → bangun API → bangun UI → pengujian → lanjut ke tabel berikutnya.
7. Alur Kerja Tim (Pengembangan Paralel)
Pembagian peran: Frontend dikerjakan oleh Walaho, backend dikerjakan oleh rekan tim. Untuk memungkinkan pengembangan paralel tanpa hambatan integrasi, tim menyepakati pendekatan contract-first.
7.1 Kontrak yang Disepakati di Awal
Prisma schema sebagai single source of truth untuk struktur data.
Bentuk API response didokumentasikan per endpoint (contoh: struktur data, nama field, format tanggal ISO 8601).
Konvensi penamaan field diseragamkan (camelCase pada response JSON).
Struktur error response diseragamkan.
Folder types/ berisi TypeScript interfaces yang digunakan bersama oleh frontend & backend.
7.2 Alur Kerja Harian
Frontend membangun UI menggunakan dummy data (mock JSON) yang mengikuti kontrak response yang telah disepakati.
Backend membangun API sesuai kontrak, diuji menggunakan Postman/Thunder Client.
Perubahan kontrak di tengah jalan harus diperbarui di dokumen kontrak bersama sebelum diimplementasikan oleh kedua pihak.
Pada tahap integrasi, pemanggilan data dummy digantikan dengan pemanggilan API sesungguhnya.
8. Isu Terbuka / Perlu Diskusi Lanjutan
Kategori dokumen PPID (jenis peraturan, notulensi, laporan keuangan, dll) belum ditentukan – menunggu diskusi dengan perangkat desa.
Detail sumber data infografis (penduduk, APBDes, stunting, bansos, IDM, SDG) belum dikonfirmasi – memengaruhi struktur kolom detail/JSONB pada tabel terkait.
Penempatan UI untuk fitur pengaduan agar tetap mudah diakses tanpa mengganggu pengalaman pengguna – perlu dirancang lebih lanjut pada tahap desain UI.
9. Catatan Teknis Tambahan
Desain mobile-first – mayoritas warga desa mengakses melalui perangkat seluler.
Preview dokumen PDF ditampilkan langsung di browser (menggunakan react-pdf), bukan hanya tautan unduhan.
Admin panel disediakan tidak hanya untuk berita, namun juga untuk konten lain (opsional, tergantung waktu pengembangan).
Aspek aksesibilitas (kontras warna, alt text gambar) dipertimbangkan namun bukan prioritas utama pada tahap ini.
